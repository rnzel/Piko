# Plan: Firestore-Safe Serialization & Sync Reliability Fix

## Problem
Firestore writes fail with **"Unsupported field value: undefined"** when Task or Group objects contain optional fields set to `undefined`. The codebase spreads these objects directly into Firestore `.set()` / `.update()` calls:

```ts
await firestore().collection(tasksPath).doc(task.id).set({
  ...task, // <-- contains undefined for optional fields (reminder?, reminderAt?, assignedTo?, etc.)
  createdBy: uid,
});
```

Firestore strictly rejects `undefined` values in document fields.

## Root Cause Analysis

### Types with optional fields (can be `undefined`)
- **Task**: `groupId?, reminder?, reminderAt?, completedAt?, completedBy?, assignedTo?, priority?, dueDate?, deleted?, lastModifiedBy?, syncVersion?, syncStatus?`
- **Group**: `description?, avatarURL?, maxMembers?, isArchived?, lastActivityAt?, taskCount?`
- **GroupMember**: `photoURL?, invitedBy?, lastActiveAt?, isOnline?, notificationsEnabled?`
- **Membership**: `lastViewedAt?, unreadCount?`

### Affected Firestore writes in `syncFacade.ts`
Every spread operation that includes a Task/Group/Member object:
1. `addTask` → `set({ ...task, ... })`
2. `updateTask` → `set({ ...task, ... })`
3. `saveTasks` → `batch.set({ ...task, ... })`
4. `uploadLocalData` → `batch.set({ ...task, ... })` and `batch.set({ ...group, ... })`
5. `addGroup` → `batch.set(groupRef, { ...group, ... })` and `batch.set(memberRef, creatorRole)` and `batch.set(membershipRef, { ... })`
6. `updateGroup` → `set({ ...group, ... }, { merge: true })`
7. `saveGroups` → `batch.set(..., group, { merge: true })`
8. `addGroupMember` → `batch.set(memberRef, member)` and membership object
9. `acceptInvitation` → `batch.set(memberRef, { ... })`
10. `logActivity` → `activityRef.set({ ...activity })`

## Solution Architecture

### 1. Centralized Sanitization Utility
Create `services/serialization.ts`:

```ts
/**
 * Recursively removes all undefined values from an object/array.
 * Firestore rejects undefined field values; this ensures only valid data is written.
 *
 * - Plain objects: keys with undefined values are deleted
 * - Arrays: each element is sanitized recursively
 * - Special objects (Date, Firebase FieldValue, etc.): returned unchanged
 */
export function sanitizeForFirestore<T>(data: T): T {
  if (data === undefined) {
    return undefined as any;
  }

  // Handle Firestore sentinel objects (e.g., FieldValue.serverTimestamp())
  // They are non-plain objects with a specific prototype/constructor
  if (
    data !== null &&
    typeof data === "object" &&
    !Array.isArray(data) &&
    data.constructor !== Object &&
    typeof (data as any).isEqual === "function" // Firestore FieldValue has isEqual
  ) {
    return data; // Firestore special value — pass through
  }

  if (Array.isArray(data)) {
    return data.map((item) => sanitizeForFirestore(item)) as any;
  }

  if (data !== null && typeof data === "object") {
    const result: Record<string, any> = {};
    for (const [key, value] of Object.entries(data as Record<string, any>)) {
      if (value !== undefined) {
        result[key] = sanitizeForFirestore(value);
      }
    }
    return result as T;
  }

  // primitives (string, number, boolean, null, symbol, bigint) pass through
  return data;
}
```

**Key design decisions:**
- Deep recursive sanitization for nested objects/arrays
- Preserves Firestore sentinel objects (e.g., `FieldValue.serverTimestamp()`) by recognizing their `isEqual` method
- Omits keys with `undefined` entirely (not set to `null`)
- Immutable: creates new objects/arrays

### 2. Apply Sanitization in `syncFacade.ts`

Import and use `sanitizeForFirestore` on **every** object passed to Firestore write methods:

- `batch.set(docRef, sanitizeForFirestore({ ... }))`
- `doc.set(sanitizeForFirestore({ ... }))`
- `doc.update(sanitizeForFirestore({ ... }))`
- For direct object parameters (like `creatorRole`): `sanitizeForFirestore(creatorRole)`

**Special handling:**
- Fields set to `firestore.FieldValue.serverTimestamp()` are preserved (passed through unmodified by sanitizer)
- No other code changes; the sanitizer is transparent

### 3. Unit Tests (Optional but recommended)
Add `serialization.test.ts` to verify:

- Removes undefined from nested objects
- Removes undefined from arrays
- Preserves `FieldValue` sentinels
- Handles null, numbers, strings safely

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Performance overhead from recursive sanitization | Firebase writes are IO-bound; sanitization is CPU-cheap. Batch operations already process in memory; overhead negligible. |
| Accidentally stripping needed fields | Sanitizer only removes `undefined`. All required fields are always defined per type contracts. Optional fields being undefined is exactly what we want to omit. |
| Breaking Firestore sentinels (e.g. `FieldValue.serverTimestamp()`)** | Explicit check for objects with `isEqual` function (Firestore sentinel signature) passes them through untouched. |
| Null values being removed | `null` is a valid Firestore value; sanitizer preserves it. Only `undefined` is removed. |
