# Plan: Make TaskModal Move with Keyboard

## Goal
Modify the TaskModal (add task modal) in `app\(tabs)\index.tsx` to properly adjust when the keyboard opens on both iOS and Android, preventing the modal from being pushed off-screen or content from being hidden.

## Current Analysis
- The modal is a bottom sheet using `Modal` with `animationType="slide"` and positioned with `justifyContent: "flex-end"`.
- It uses `KeyboardAvoidingView` with `behavior={Platform.OS === "ios" ? "padding" : undefined}`, so on Android, no adjustment is made.
- On Android, when keyboard opens, the modal gets pushed up, potentially moving off-screen or resizing incorrectly.
- On iOS, it adds padding to adjust, but may not be sufficient.

## Proposed Solution
1. Enable keyboard adjustment on Android by setting `behavior` to `"height"` or `"padding"`.
2. Move the `KeyboardAvoidingView` inside the modal to wrap the `modalSheet` View, so only the content adjusts, not the modal position.
3. Adjust `keyboardVerticalOffset` for better positioning.
4. Test on both platforms to ensure the modal "moves" (adjusts) with the keyboard without being pushed away.

## Implementation Steps
1. Read the current modal code around lines 559-683.
2. Modify the `KeyboardAvoidingView` props:
   - Set `behavior={Platform.OS === "ios" ? "padding" : "height"}`
   - Adjust `keyboardVerticalOffset` if needed (e.g., set to 0 or a small value).
3. Move the `KeyboardAvoidingView` to wrap the `View` with style `modalSheet` instead of the outer `View`.
4. Verify with lint and test that the modal content scrolls or adjusts when keyboard opens.

## Questions for User

- Are there any issues on iOS currently, or is it only Android?
