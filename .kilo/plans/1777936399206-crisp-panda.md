# Plan: Make Tasklist Full Width

## Objective
Adjust the tasklist to span 100% of the container width by removing left/right margins from task items, without disrupting existing styles or affecting other elements.

## Current State Analysis
- **File**: `app/(tabs)/index.tsx`
- **Container** (`styles.container`): has `paddingHorizontal: 20` — this will remain unchanged as per user clarification.
- **Task List** (`styles.taskList`): white ScrollView container with `borderRadius: 24` and `paddingTop: 10`.
- **Task Items** (`styles.taskItem`): individual task rows with `marginHorizontal: 16` on line 873 — this creates the unwanted side margins.

## Planned Changes
1. **Remove horizontal margins from task items**
   - Delete `marginHorizontal: 16` from the `taskItem` style object (line 873).

2. **Adjust task item internal padding (optional but recommended)**
   - Consider increasing the left/right `padding` from `16` to `16` if needed to maintain visual density, but keeping it unchanged preserves existing design spacing between content and borders.

3. **Verify no other elements depend on those margins**
   - Task items have `borderRadius: 16` - without horizontal margins, multiple tasks in a row will appear butted up against each other, which is fine. The white background and border will show. The visual gap between items will still be preserved by `marginBottom: 10`.
   - The task list background is white with rounded corners, so edge tasks won't have extra margin space.

4. **Test**
   - Visually confirm that:
     - Task items now extend edge-to-edge within the white tasklist container.
     - The white tasklist container's `borderRadius: 24` remains correct at the edges.
     - No layout shift occurs for header, stats, filter, or floating button.
     - Touch targets remain adequate.
